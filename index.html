FAQ

Common questions

Does my PDF really never leave my device?

Yes. All processing uses JavaScript and WebAssembly running inside your browser tab. You can verify this yourself: open DevTools (F12) → Network tab → process a PDF and watch zero outgoing requests appear. The privacy claim is independently auditable.

Why is the free tier unlimited tasks with just a file size cap?

Unlike competitors (Smallpdf limits 2 tasks/hour, iLovePDF caps files at 15 MB), we have no per-task compute costs because processing is client-side. Our only real constraint is that very large files can be slow in-browser — hence the 25 MB cap on the free tier.

Is this actually GDPR / HIPAA compliant?

Our architecture satisfies GDPR Article 25 (data minimisation) by design — we collect no personal data because none passes through our infrastructure. For HIPAA, because no PHI ever touches our servers, you're operating within your own environment. No BAA or DPA is required.

How is this 90% cheaper than Adobe?

Adobe Acrobat Pro is $19.99–$29.99/user/month. Our Pro plan is $1/month. The Team plan is $8/seat/month. Because we have zero per-user server costs (~85% gross margin), we can price far below the market while remaining profitable.

What's on the roadmap?

PDF compression, e-signatures, a Chrome extension for one-click PDF actions, client-side AI features (summarise, extract data via WebLLM), and a JavaScript SDK for developers to embed tools into their own products. These features will be available to Pro and Team users first.

Do you plan to open-source the PDF processing layer?

Yes. Publishing the core processing code on GitHub is on the roadmap as a trust signal for enterprise buyers and privacy advocates. The billing and auth infrastructure will remain closed-source.
